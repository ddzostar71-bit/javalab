# Инструкция для демонстрации преподавателю

Проект настроен для автоматической проверки через GitHub Actions. Ниже несколько способов показать работу.

## Способ 1: Badge в README

На главной странице репозитория есть badge (зеленый значок). Если он зеленый - всё работает.

## Способ 2: Просмотр логов выполнения

1. Открыть вкладку **Actions** на GitHub
2. Выбрать последний запуск из списка
3. Нажать на задачу "Сборка и тестирование проекта"
4. Развернуть нужные шаги для просмотра логов

В шаге "Запуск главного класса" будет виден полный вывод программы с результатами работы всех методов.

## Способ 3: Скачать отчет (рекомендуется)

1. На странице запуска прокрутить вниз до раздела **Artifacts**
2. Скачать ZIP архив с отчетом
3. Открыть файл `CONTROL_WORK_REPORT.txt`

В отчете:
- Дата и время выполнения
- Описание задания варианта 5
- Список всех требуемых методов
- Вывод программы с демонстрацией работы
- Результаты тестирования

## Способ 4: Запуск вручную

Можно запустить тестирование при преподавателе:

1. Открыть **Actions**
2. Выбрать "Тестирование контрольной работы"
3. Нажать **Run workflow**
4. Выбрать ветку и запустить

Через 2-3 минуты будут готовы результаты.

## Способ 5: Интерактивный режим

Есть второй пайплайн "Интерактивное тестирование" с возможностью указать параметры:
- ID для поиска
- ID для удаления
- Данные для создания нового туриста
- Направление для обновления

## Что показать преподавателю

1. Открыть репозиторий на GitHub
2. Показать зеленый badge
3. Перейти в Actions и показать успешные запуски
4. Открыть последний запуск и показать логи
5. Скачать и открыть отчет из Artifacts

Отчет уже оформлен и содержит всю необходимую информацию.
